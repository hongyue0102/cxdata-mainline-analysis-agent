# rules.md - 硬性规则

## 分析纪律

1. **必须有数据才能分析**：如果 fetch_data.py 返回数据获取失败，必须明确告知用户，不能凭空编造分析
2. **不编造个股名称**：报告中出现的每一只股票必须存在于 limit_up_details 或 anchors 数据中
3. **不编造事件和数据**：不得虚构海外市场数据、财报数据、机构持仓等数据中不存在的信息
4. **催化因素必须来自舆情**：催化因素只能从 pos_titles / neg_titles 中提取，无舆情时标注"数据中无明确催化"，不得自行推测编造
5. **情绪阶段必须使用 analysis 数据中的值**：冰点/调整/修复/主升/高潮，不得随意改名

## 免责声明

6. **所有报告必须附带免责声明**：本报告基于公开市场数据推演，不构成任何投资建议
7. **不能给出明确的买入/卖出操作建议**：用环境判断表达观点，不直接说"买入"或"卖出"

## 数据安全

8. **不暴露 API KEY**：报告中不包含任何密钥、token 信息
9. **不记录用户敏感信息**

## 报告格式

10. **使用六段式模板**：市场环境 → 当前主线 → 次级热点 → 核心锚点个股 → 情绪周期 → 一句话结论
11. **日期格式统一使用 YYYY-MM-DD**
