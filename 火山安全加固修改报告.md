## 二、详细变更（与 stock agent 加固内容一致）

### 2.1 auth.py — 审计日志 + 拒绝协议确认机制

- 新增审计日志系统（`_audit_scrub` / `_audit` / `_audit_logger`）
- `_mask_phone` 短号码保护（长度 ≤ 7 返回 `****`）
- `cmd_terms_decline` 增加三重确认机制（`--yes` / stdin JSON / 交互终端）
- HTTP 请求加 `allow_redirects=False`

### 2.2 common.py — 安全纵深加固

- authtoken 加密存储 + 透明解密
- `_safe_env_executable` 支持多目录白名单
- `_python_trusted_dirs()` — CXDA_CACHE_PYTHON RCE 防护
- `_is_valid_user_key` 格式校验
- 纯点文件名拒绝（`_ALL_DOTS_RE`）
- 新增 `http_post_form()` + `_validate_official_url()` + `_decode_response()`
- `fetch_new_token()` 改为 POST + verify=True + allow_redirects=False
- gzip 炸弹防护（32MiB 上限）
- `http_get()` 加 verify=True + allow_redirects=False

### 2.3 cred_crypto.py — 密钥派生大幅加固

- 引入 pepper 文件（`~/.cxda-cache/.cred_pepper.bin`，O_NOFOLLOW + O_EXCL）
- `_get_effective_username()` 替代 `getpass.getuser()`
- 密钥材料改用 JSON 序列化防碰撞
- 三项特征（host+user+machine_id）均不可空
- 解密异常细分

### 2.4 cxda_cache_cli.py — symlink 攻击防护

- 新增 `_ensure_dir_not_symlink()` — 拒绝 workspace 为符号链接
- `/root` 从关键目录黑名单移除
- 新增 `_secure_read_text()` — O_NOFOLLOW 读取
- `_secure_write_text()` 加 O_NOFOLLOW
- delete 操作改用词法路径（不 resolve）
- 公域/私域文件名校验增强 + 纯点名拒绝

### 2.5 query.py — 跟随 common.py 改动

- `import http_post_form`
- `_query_package_result` 和 `_fetch_api_limit_setting` 中 userKey 从 GET params 改为 POST data
- API ID 路径注入校验（`_API_ID_RE`，已有）

### 2.6 requirements.txt

- `cryptography>=42.0.0` → `cryptography>=42.0.0,<45.0.0`
- 新增 `requests>=2.32.0,<3.0.0`
